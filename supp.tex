\section{Theoretical Derivation of Token Compression}
\label{sec:supp_derivation}

为说明所提出 token 压缩框架（以 \textit{BlackHoleMerge} 为例）的结构完整性、排序
适配机制及计算收益，下面给出严格推导。需要强调的是，以下结论证明的是算子的
非空性、完整分配、K-norm 排序重排条件及局部扰动上界；下游 OCR/文档解析精度的
保持属于经验命题，应由正文中的匹配训练与评测结果验证。

\subsection{K-norm 密度与 token 选择}

设第 $\ell$ 个视觉 Transformer 层的输入 token 为
$\{x_i\}_{i=1}^{N}$，其中 $x_i\in\mathbb{R}^{d_x}$。对应的 attention key 为
\begin{equation}
    k_i=W_Kx_i+b_K\in\mathbb{R}^{d_k},\qquad
    r_i=\lVert k_i\rVert_2 .
    \label{eq:key_norm}
\end{equation}
对同一图像内的 $N$ 个 token，定义
\begin{equation}
    \mu=\frac{1}{N}\sum_{t=1}^{N}r_t,\qquad
    \sigma=\sqrt{\frac{1}{N}\sum_{t=1}^{N}(r_t-\mu)^2},
    \label{eq:key_statistics}
\end{equation}
并以 z-score 归一化 K-norm 后得到 density
\begin{equation}
    z_i=\frac{r_i-\mu}{\sigma+\epsilon},\qquad
    d_i=\operatorname{sigmoid}(z_i).
    \label{eq:density}
\end{equation}
令 $S=\{i\mid d_i\geq 1/2\}$ 为保留的 absorber 集合，
$R=\{1,\dots,N\}\setminus S$ 为待吸收集合。由于 sigmoid 单调且
$\operatorname{sigmoid}(0)=1/2$，有
\begin{equation}
    d_i\geq\frac12\Longleftrightarrow z_i\geq0
    \Longleftrightarrow r_i\geq\mu.
    \label{eq:threshold_equivalence}
\end{equation}
因此，BlackHoleMerge 按 token 的相对 K-norm，而非绝对尺度进行选择。

\paragraph{相对排序的平衡性。}
由式~\eqref{eq:key_statistics}--\eqref{eq:density} 可得
\begin{equation}
    \sum_{i=1}^{N}z_i=
    \frac{1}{\sigma+\epsilon}\sum_{i=1}^{N}(r_i-\mu)=0.
    \label{eq:zscore_sum_zero}
\end{equation}
所以只要存在一个 $z_i>0$，必然存在至少一个 $z_j<0$。这给出了当某个视觉token的density变大导致其排名上升时，必然会有其他视觉token排名下降的严格含义：它描述的是相对于样本内均值的排序位置，而非所有K-norm 的绝对数值。特别地，若所有 $r_i$ 仅作相同的正仿射变换，z-score 的排序不变；有效的压缩适配必须来自 token 间非均匀的 K-norm 更新。

\subsection{K 空间分配与结构完整性}

对每个被吸收 token $j\in R$，定义其 absorber 为
\begin{equation}
    a(j)=\arg\max_{i\in S}
    \left[\frac{k_j^\top k_i}{\lVert k_j\rVert_2\lVert k_i\rVert_2}d_i\right].
    \label{eq:assignment}
\end{equation}
对固定的 $S$，式~\eqref{eq:assignment} 最大化如下可分目标：
\begin{equation}
    \max_{a:R\rightarrow S}\sum_{j\in R}
    \operatorname{cos}(k_j,k_{a(j)})d_{a(j)}.
    \label{eq:assignment_objective}
\end{equation}
证明很直接：式~\eqref{eq:assignment_objective} 对每个 $j$ 可分，对每个 token
单独取最大 attraction 即得到全局最优分配。由 $\arg\max$ 的唯一输出规则，
$\{j\in R:a(j)=i\}_{i\in S}$ 构成 $R$ 的不交划分，因而不存在漏分配或重复分配。

设 $h_i\in\mathbb{R}^{d}$ 是用于下游 projector/LLM 的视觉特征。部署的残差吸收为
\begin{equation}
    \widetilde h_i=h_i+
    \sum_{j\in R:a(j)=i}d_j(h_j-h_i),\qquad i\in S.
    \label{eq:residual_merge}
\end{equation}
因此每个 absorber 的局部扰动满足
\begin{equation}
    \left\lVert\widetilde h_i-h_i\right\rVert_2
    \leq\sum_{j\in R:a(j)=i}d_j\left\lVert h_j-h_i\right\rVert_2.
    \label{eq:local_bound}
\end{equation}
式~\eqref{eq:local_bound} 由三角不等式直接得到。它说明低 density token 的影响被
显式衰减；同时，式~\eqref{eq:assignment} 倾向于将其分配给 K 空间中相似的 absorber，
从而减小残差距离。

需要注意，式~\eqref{eq:residual_merge} 不是普通加权均值，通常不守恒未加权 feature
sum。因此本文的严格结论是``density 加权残差受到控制''，而非``视觉特征严格守恒''。

\subsection{CE 对 K-norm 排序的间接驱动}

本节回答一个核心问题：BlackHoleMerge 中的 hard threshold 和 $\arg\max$ 均不可微，
那么在只使用 teacher-forcing CE 的 SFT 阶段，为什么 K-norm 的排序仍会逐步改变？
答案是，CE 不直接对 density 求导；它首先改变视觉 attention 的 key 表示，再由
key 的模长变化改变 K-norm 排序。完整链路为
\begin{equation}
    \mathcal{L}_{\mathrm{CE}}
    \longrightarrow \text{attention logits}
    \longrightarrow k_i
    \longrightarrow r_i=\lVert k_i\rVert_2
    \longrightarrow z_i
    \longrightarrow \text{keep/absorb decision}.
    \label{eq:ce_ranking_chain}
\end{equation}

为避免符号混淆，先说明本节记号。下标 $a$ 表示发出查询的 token 位置，下标 $b$
表示作为候选 key/value 的 token 位置；$q_a,k_b,v_b$ 分别是该 attention head 中
位置 $a,b$ 的 query、key、value；$s_{ab}$ 是 $a$ 对 $b$ 的 attention logit；
$\alpha_{ab}$ 是 softmax 后的 attention 权重；$y_a$ 是 attention 输出。令
$g_a=\partial\mathcal{L}_{\mathrm{CE}}/\partial y_a$，即从下游 projector、LLM
和 CE 反传到该 attention 输出的梯度。

对单个 attention head，有
\begin{equation}
    s_{ab}=\frac{q_a^\top k_b}{\sqrt d},\qquad
    \alpha_{ab}=\frac{\exp(s_{ab})}{\sum_c\exp(s_{ac})},\qquad
    y_a=\sum_b\alpha_{ab}v_b.
    \label{eq:attention}
\end{equation}
下面分三步求 CE 对 key 的梯度。首先由注意力输出 $y_a=\sum_b\alpha_{ab}v_b$，利用链式法则分解梯度：
\begin{equation}
\begin{aligned}
\frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial\alpha_{ab}}
&= \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial y_a}
\cdot \frac{\partial y_a}{\partial \alpha_{ab}} \\
&= g_a^\top v_b.
\end{aligned}
\label{eq:ce_to_attention_weight}
\end{equation}

其次，softmax 将一组输入相似度分数 $\{s_{a1},s_{a2},\dots\}$ 映射为归一化注意力权重 $\{\alpha_{a1},\alpha_{a2},\dots\}$。任意一个输入 logit $s_{ab}$ 发生扰动时，全体输出权重 $\alpha_{ac}$ 都会随之改变，因此雅可比矩阵元素描述任意输出权重 $\alpha_{ac}$ 对输入分数 $s_{ab}$ 的偏导：
\begin{align}
    \frac{\partial\alpha_{ac}}{\partial s_{ab}}
    =\alpha_{ac}\bigl(\mathbb{I}[b=c]-\alpha_{ab}\bigr).
    \label{eq:softmax_jacobian}
\end{align}
式中：固定 $a$ 为同一Query；$b$ 是被扰动的输入logit下标；$c$ 遍历全部输出权重，为求和哑标；$\mathbb{I}[b=c]$ 为指示函数，仅当 $c=b$ 取1，其余取0。
根据多元链式法则，损失对目标 logit $s_{ab}$ 的梯度需要累加所有权重 $\alpha_{ac}$ 的传导贡献：
\[
\frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial s_{ab}}
=\sum_{c} \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial\alpha_{ac}} \cdot \frac{\partial\alpha_{ac}}{\partial s_{ab}}.
\]
将式~\eqref{eq:ce_to_attention_weight} 与式~\eqref{eq:softmax_jacobian} 代入求和并分项化简，结合指示函数与注意力输出定义 $y_a=\sum_c\alpha_{ac}v_c$ 合并项后，得到
\begin{equation}
    \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial s_{ab}}
    =\alpha_{ab}g_a^\top(v_b-y_a).
    \label{eq:ce_to_logit}
\end{equation}

最后，注意力 logit 的定义为 $s_{ab}=q_a^\top k_b/\sqrt{d}$，我们需要通过链式法则，将损失对 logit $s_{ab}$ 的梯度传导至 Key 向量 $k_b$。
对于固定的一组 $(a,b)$，先求标量 $s_{ab}$ 对向量 $k_b$ 的梯度：
\[
\frac{\partial s_{ab}}{\partial k_b} = \frac{1}{\sqrt{d}} q_a.
\]
损失 $\mathcal{L}_{\mathrm{CE}}$ 依赖全部 logit $\{s_{a1},s_{a2},\dots\}$，因此对向量 $k_b$ 的梯度需要遍历所有 Query 下标 $a$ 累加贡献：
\[
\frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial k_b}
=\sum_{a} \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial s_{ab}} \cdot \frac{\partial s_{ab}}{\partial k_b}.
\]
将前文求得的 $\frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial s_{ab}}=\alpha_{ab}g_a^\top(v_b-y_a)$ 与 $\partial s_{ab}/\partial k_b$ 一并代入求和式：
\begin{equation}
    \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial k_b}
    =\frac{1}{\sqrt d}\sum_a
    \alpha_{ab}\left[g_a^\top(v_b-y_a)\right]q_a.
    \label{eq:ce_to_key}
\end{equation}
式~\eqref{eq:ce_to_key} 表明，只要 token $b$ 在任意 Query $a$ 的注意力计算中存在贡献、能够影响下游输出概率，交叉熵损失一般会向该 token 的 Key $k_b$ 传回非零梯度。不同视觉 token 对应的注意力权重 $\alpha_{ab}$、Value $v_b$、Query $q_a$ 以及上层误差信号 $g_a$ 均存在差异，使得各个 token 的 Key 更新梯度互不相同；这种差异化梯度是训练过程中 K-norm 排序发生重排的根本来源。

接下来我们把求得的 Key 梯度进一步转化为 K-norm $r_i=\|k_i\|_2$ 的变化量。
记 $g_i^K = \partial\mathcal{L}_{\mathrm{CE}}/\partial k_i$ 为 CE 损失对第 $i$ 个视觉 token Key 向量的梯度。
需要注意：全部视觉 token 共用同一套 Key 投影参数 $W_K$、偏置 $b_K$，各个 $k_i$ 并非相互独立的可更新变量，任意 token 的梯度都会耦合影响其余所有 token 的 Key。

标准 SGD 参数更新规则为
\begin{equation}
    W_K^+=W_K-\eta\sum_{t=1}^{N}g_t^Kx_t^\top,\quad
    b_K^+=b_K-\eta\sum_{t=1}^{N}g_t^K.
    \label{eq:sgd_wk}
\end{equation}
其中 $\eta$ 代表学习率，$N$ 为单张图像内视觉 token 总数。将更新后的 $W_K^+,b_K^+$ 代入 Key 定义 $k_i^+=W_K^+x_i+b_K^+$，做代数展开整理，可解出单次迭代内 token $i$ 的 Key 一阶变化量：
\begin{equation}
    k_i^+-k_i=-\eta G_i^K,\quad
    G_i^K=\sum_{t=1}^{N}g_t^K(x_t^\top x_i+1).
    \label{eq:induced_key_update}
\end{equation}
耦合项 $G_i^K$ 的物理含义为：图像内所有 token 的梯度，通过共享投影矩阵共同作用，最终给 token $i$ 带来的等效更新方向。

定义单位径向向量 $u_i = \dfrac{k_i}{\|k_i\|_2}$，代表 $k_i$ 模长增长/收缩的基准方向。
对模长函数 $r_i=\|k_i\|_2$ 在当前 $k_i$ 处做一阶泰勒近似，代入 Key 更新量 $k_i^+-k_i=-\eta G_i^K$，忽略二阶小项 $O(\eta^2)$ 后得到模长变化：
\begin{equation}
    r_i^+-r_i=-\eta u_i^\top G_i^K+O(\eta^2).
    \label{eq:norm_update}
\end{equation}
该式可直观判断单步更新中 K-norm 的升降趋势：
- 若 $u_i^\top G_i^K < 0$：模长变化量 $r_i^+-r_i>0$，$r_i$ 增大；
- 若 $u_i^\top G_i^K > 0$：模长变化量 $r_i^+-r_i<0$，$r_i$ 减小。

结合前文结论式~\eqref{eq:zscore_sum_zero}，整张图像所有 token 的 z-score 变化总和恒为0。只要本轮参数更新带来了非均匀的模长调整，图像内必然同时出现相对均值上升、相对均值下降的 token，这就为下一轮前向传播时发生 K-norm 排名翻转、keep/absorb 阈值切换提供了前提条件。

\subsection{排序与阈值翻转条件}

我们先分析两个 token 之间 K-norm 排名发生互换的条件。
设当前满足 $r_i>r_j$，定义二者的排序余量 $m_{ij}=r_i-r_j>0$，代表当前 token $i$ 相较于 $j$ 的模长优势。
将式~\eqref{eq:norm_update} 分别代入 $r_i$、$r_j$ 的更新规则，作差得到更新后的排序余量：
\begin{equation}
    m_{ij}^+=m_{ij}-\eta(u_i^\top G_i^K-u_j^\top G_j^K)+O(\eta^2).
    \label{eq:rank_margin}
\end{equation}
忽略高阶无穷小 $O(\eta^2)$，若更新后 token $j$ 的 K-norm 反超 $i$，则需要满足 $m_{ij}^+ < 0$，整理得到排名翻转的一阶充分条件：
\begin{equation}
    \eta(u_i^\top G_i^K-u_j^\top G_j^K)>m_{ij}.
    \label{eq:rank_crossing}
\end{equation}
该式直观说明：当两个 token 的径向梯度差异乘以学习率，能够抵消甚至超过原本的模长差距时，二者的 K-norm 排序就会发生翻转。

接下来分析单个 token 跨越保留/吸收阈值的条件。
定义阈值余量 $d_i=r_i-\mu$，代表当前 token $i$ 的 K-norm 相对整张图像平均 K-norm 的偏移量；$d_i>0$ 代表该 token 会被保留，$d_i<0$ 则会被吸收。
图像均值的更新量满足 $\Delta\mu=\frac{1}{N}\sum_t \Delta r_t$，结合模长更新公式式~\eqref{eq:norm_update}，可以推导出阈值余量的变化量：
\begin{equation}
    d_i^+-d_i=-\eta\left[u_i^\top G_i^K-
    \frac{1}{N}\sum_tu_t^\top G_t^K\right]+O(\eta^2).
    \label{eq:threshold_margin}
\end{equation}
括号内两项的差值表征：token $i$ 自身径向更新，对比全图所有 token 的平均径向更新水平。
若某 token 当前处于被吸收状态 $d_i<0$，更新后变为保留 token 的一阶充分条件为 $d_i^+>0$，整理得：
\begin{equation}
    -\eta\left[u_i^\top G_i^K-
    \frac{1}{N}\sum_tu_t^\top G_t^K\right]>-d_i.
    \label{eq:threshold_crossing}
\end{equation}
上式物理含义为：只有自身径向更新幅度高于图像整体平均更新幅度的 token，其相对均值的偏移量 $d_i$ 才会正向抬升，存在跨过阈值、从被吸收转为保留的可能。

\subsection{硬压缩的不可微性与三阶段训练的必要性}
当前工程部署的 BlackHoleMerge 包含三处阻断梯度的离散操作：硬阈值判别、$\arg\max$ 硬分配，同时对被吸收 token 的 density 权重执行梯度截断（stop-gradient）。在不跨越保留/吸收离散边界的局部区间内，合并输出特征对密度分数的导数恒为0，进而损失无法经由压缩分支传回密度打分：
\begin{equation}
    \frac{\partial\widetilde H}{\partial p_i}=0,\quad
    \frac{\partial\mathcal{L}_{\mathrm{CE}}}{\partial d_i}
    \bigg|_{\mathrm{merge\ branch}}=0.
    \label{eq:hard_merge_gradient}
\end{equation}
上式说明交叉熵损失无法通过压缩算子支路直接驱动 token 的 density 提升至判定阈值之上。CE 的唯一适配路径为式~\eqref{eq:ce_ranking_chain} 给出的原生视觉 Attention Key 通路，仅能连续调整 Key 表征；只有当参数更新满足式~\eqref{eq:rank_crossing} 排序翻转条件或式~\eqref{eq:threshold_crossing} 阈值跨越条件时，下一轮前向传播才会切换离散的保留/吸收掩码。

该不可微特性天然划分了三阶段训练各自的核心职责：监督微调（SFT）仅依靠 CE 损失优化连续视觉表征，保证基础文档解析性能；强化学习阶段（GRPO）引入可微的 DensityPolicy 策略头，直接优化离散压缩策略，平衡压缩率与任务保真度；后微调（post-SFT）固定训练完成的压缩策略，重新适配投影层与大语言模型，使其适应缩短后的视觉 Token 输入上下文。

\subsection{计算复杂度削减分析}
原生自注意力机制具备二次计算复杂度 $\mathcal{O}(N^2 D)$。我们在中间瓶颈层 $l_{mid}$ 引入本文压缩框架后，整体总计算开销 $C_{total}$ 由原始完整计算曲线变为分段递减形式：
\begin{equation}
C_{total} = \sum_{l=1}^{l_{mid}-1} \mathcal{O}(N^2 D) + \sum_{l=l_{mid}}^{L} \mathcal{O}(M^2 D)
\end{equation}
将压缩后 Token 数量 $M = \gamma N$ 代入后半段求和式，可量化得到计算量节省比例 $\Omega$：
\begin{equation}
\Omega = 1 - \frac{(l_{mid}-1) + \gamma^2(L - l_{mid} + 1)}{L}
\end{equation}
以深度视觉大模型（VLM）为例，设置总层数 $L=32$、启用压缩的中间层 $l_{mid}=8$、压缩比例 $\gamma=0.4$，代入计算可得视觉主干网络浮点运算量（FLOPs）降低约 \textbf{59.3\%}。

\subsection{Training Ablation Experiments}
PaddleOCR-VL-1.6 的完整系统采用 coarse-to-fine pipeline。前端版面/区域模块首先在整页图像上定位有效区域，并过滤页边距、空白和背景区域；随后，裁剪后的局部高分辨率区域被送入 0.9B VLM 组件进行 OCR 与结构化解析。因此，官方完整系统中的一部分冗余消除发生在 VLM 之前，依赖外部区域选择模块，而不是 VLM 内部的视觉 token 压缩。

在附录中，我们重点分析 BlackHole Merge 在 VLM 视觉 token 路径内部的训练消融。具体而言，我们从 PaddleOCR-VL-1.6 中取出 0.9B VLM 组件，并构建去除外部裁剪模块的 end-to-end 设置：整页图像直接输入视觉编码器，视觉 token 的保留与合并由压缩模块决定。为避免原始 $4\times$ projector 的固定空间压缩与本文方法混合，我们使用 $1\times$ per-token projector，将 ViT hidden size 从 1152 映射到 LLM hidden size 1024，并不再执行额外的 $2\times2$ spatial merge。这样，不同变体之间的差异主要来自训练路径和 token 压缩策略。

我们在相同 PaddleOCR-VL-1.6B VLM 初始化、相同 $1\times$ projector、相同训练数据和相同评测协议下比较四种训练路径。\textbf{No-merge SFT} 不启用 BlackHole Merge，仅进行普通 SFT，用作无压缩参考和共享 backbone。\textbf{Merge-aware SFT only} 从 SFT 阶段启用 BlackHole Merge，使 ViT、projector 和 LLM 共同适应压缩后的视觉 token 分布。\textbf{RL only} 不经过 merge-aware SFT，直接训练轻量 policy MLP 进行 keep/drop 决策，用于验证 RL 是否能够在没有稳定 density 排序先验的情况下独立学习压缩策略。\textbf{Merge-aware SFT + Confidence-Penalty RL} 则先通过 SFT 学习粗粒度 density 排序，再冻结 ViT、projector 和 LLM，仅训练轻量 policy MLP，对 $0.5$ 附近的 keep/drop 边界进行细化。

Tables~\ref{tab:token_density} and~\ref{tab:keep_ratio} summarize the density and keep-ratio ablations. Merge-aware SFT enlarges the density gap between text and blank/background tokens, indicating that the model learns to assign higher density to text, formula, and structural regions while suppressing blank and background regions. In contrast, RL only can reduce the overall keep ratio, but without the SFT ranking prior it tends to remove more text tokens, leading to worse parsing quality than SFT-only. This confirms that RL should not be used as an independent compression learner from scratch, but rather as a post-SFT boundary refinement stage.

经过 Confidence-Penalty RL 后，density 分布进一步极化：blank/background token 的 density 被显著压低，gray zone 明显缩小，SFT 得到的连续排序被转化为更清晰的 keep/drop 决策。这组消融说明，SFT 负责形成任务相关的 density 排序，RL 负责在该排序附近学习更激进但受质量约束的压缩边界。

MinerU2 的完整文档解析系统同样采用 pipeline 形式：前端模块根据页面布局和任务类型选择待解析区域或页面块，随后由 VLM 对相应区域执行文本、公式和版面结构解析。因此，原始系统的一部分冗余消除也发生在 VLM 之前，依赖外部区域选择与裁剪，而不是 VLM 内部的视觉 token 压缩。

为验证 BlackHoleMerge 是否依赖单一模型结构，我们从 MinerU2 中单独抽取其 0.9B VLM 组件，并构建不依赖前端区域裁剪的端到端 VLM 设置。输入页面不经过外部区域选择或多阶段调度，而是直接进入 VLM 视觉编码器。该设置测量的是视觉主干内部 token 压缩的作用，而非 pipeline 前端区域筛选带来的计算节省。

MinerU2-VLM 的视觉塔为 26 层 SigLIP 编码器，视觉 hidden size 为 1152。按照该 VLM 组件的单 tile 输入配置，每个页面被直接 resize 为一个 $384\times384$ 的视觉 tile，并产生
\begin{equation}
N_0 = 27 \times 27 = 729
\end{equation}
个视觉 token。视觉特征随后经逐 token 的 MLP projector 映射到语言模型的 hidden size 896。该 projector 不执行额外的空间合并，因此视觉 token 数量的变化只由 BlackHoleMerge 决定。

我们将 BlackHoleMerge 插入 SigLIP 第 20 层之后。前 20 个视觉 Transformer blocks 首先在完整的 729-token 序列上运行；在第 20 层中，以该层 attention key 的范数构造 density score，并据此选择 absorber token。低 density token 被分配给相似度最高的 absorber，并以 density 加权残差注入该 absorber。压缩后的短序列继续经过第 21--25 层 SigLIP blocks，最后送入 projector 和语言模型。
因此，合并并非发生在完整视觉编码器之后。第 21--25 层的 self-attention 已直接在缩短后的视觉序列上计算；较短的视觉序列也相应降低 projector 计算和语言模型 prefill 阶段的视觉上下文长度。

与固定 token 数压缩不同，MinerU2 中的保留数量由每个输入 tile 的 K-norm density 分布自适应决定。对于同一训练 batch 内的不同视觉 tile，729 个输入 token 可被动态压缩至不同长度。例如，实际训练日志中同一 batch 的 8 个 tile 分别被压缩为 418、434、456、455、419、434、412 和 401 个 token。包含更密集文字、公式或复杂版面的页面会保留更多 token，而背景、空白或低信息区域会被更强地吸收。因此，压缩率由页面内容决定，而不是预先指定固定保留比例。

为保证比较公平，no-merge baseline 与 BlackHoleMerge 模型采用相同的 MinerU2-VLM 初始化、输入分辨率、SigLIP 主干、逐 token projector 和语言模型。no-merge 设置保留完整的 729 个视觉 token；BlackHoleMerge 设置仅在第 20 层后引入自适应合并。两者的性能与计算差异因而可以归因于 VLM 内部的 token 合并，而不是外部区域筛选、裁剪策略或 projector 架构变化。

该适配说明 BlackHoleMerge 不依赖 PaddleOCR-VL 特定的视觉编码器或其 coarse-to-fine pipeline。即使在 MinerU2 这一具有不同视觉塔、projector 和语言模型 hidden size 的 VLM 组件中，attention key 表征仍提供可用于内容自适应压缩的 density 信号。通过在视觉编码器中间层执行合并，冗余消除能够从系统 pipeline 前端转移到 VLM 内部，并同时减少后续 ViT、projector 和 LLM prefill 的计算负担。


As an OCR-centric VLM, dots.ocr features tight coupling between text layouts and vision tokens, making it highly sensitive to uniform token pruning. We modify its dense feature map projection layer to support variable-length compressed token inputs instead of static grids. This structural adjustment allows our merging algorithm to wrap closely around dense content blocks in multi-column layouts. The adapted version achieves a \textbf{40.0\% token savings} without suffering from character-level information loss or layout fragmentation.
